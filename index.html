Hello, friend. Welcome. Mutation Observer. I recently learned about the 
MutationObserver
interface. I currently use it like I would use a normal 
EventListener
, however, this lets me react to something (e.g. a changing HTMLElement 
attribute
) rather than to a normal event. I like this mainly because I am already keeping UI component state in CSS classes. I can observe a targetNode for mutations (where 
mutation.type="attribute"
) and then check for the particular CSS 
--is-state
class I'm looking for.

Here is the snippet of code that I use to dim the opacity of the main content area of this page if the navigation menu has it's "open" class applied: 
const targetElemObserver = new MutationObserver((mutations) => {
  for(let i = 0; i<mutations.length; i++){
    const mutation = mutations[i];
    if(mutation.type === 'attributes' && mutation.attributeName === 'class'){
      if(dimWhenOpenTargetElem.classList.contains(openClassName) && !elem.classList.contains(dimClass)){
        elem.classList.add(dimClass);
      }
    }
  }
});
targetElemObserver.observe(dimWhenOpenTargetElem, { attributes: true });

TL;DR
  1. Let's me observe changes to an HTMLElement's attributes (rather than listening for particular events).
  2. Let's me decouple logic that should execute when an element is in a particular state from the events that might put said element into that state. (If I keep track of element state via CSS 
    --is-state
    classes).
